C++ काय आहे?

C++ ही एक सामान्य उद्देशाने वापरली जाणारी प्रोग्रामिंग भाषा आहे आणि सध्या स्पर्धात्मक प्रोग्रामिंगसाठी मोठ्या प्रमाणावर वापरली जाते. यात इम्पेरिटिव्ह, ऑब्जेक्ट-ओरिएंटेड आणि जनरिक प्रोग्रामिंग वैशिष्ट्ये आहेत. C++ विविध प्लॅटफॉर्म्सवर चालते जसे की Windows, Linux, Unix, Mac इत्यादी. C++ ही एक कार्यक्षम आणि शक्तिशाली भाषा आहे आणि विविध GUI प्लॅटफॉर्म्स, 3D ग्राफिक्स आणि रिअल-टाइम सिम्युलेशन्समध्ये मोठ्या प्रमाणावर वापरली जाते. समृद्ध फंक्शन लायब्ररींच्या समावेशामुळे C++ मध्ये काम करणे C पेक्षा सोपे आणि सोयीचे होते. Java प्रमाणेच, C++ ऑब्जेक्ट-ओरिएंटेड प्रोग्रामिंगला समर्थन देते आणि विरासत, पॉलीमॉर्फिझम, एन्कॅप्सुलेशन इत्यादी वैशिष्ट्ये प्रदान करते. C पेक्षा वेगळे, C++ मध्ये एक्सेप्शन हँडलिंग आणि फंक्शन ओव्हरलोडिंगची सुविधा आहे.

“Hello World” हा प्रोग्राम शिकण्याच्या सुरुवातीचा टप्पा आहे आणि तुम्ही शिकणारा सर्वात सोपा प्रोग्राम आहे. तुम्हाला फक्त "Hello World" हा संदेश स्क्रीनवर प्रदर्शित करायचा आहे.

आता प्रोग्राम पाहूया:

#include
using namespace std;
int main()
{
cout<<"Hello World";
return 0;
}

C++ ही एक ऑब्जेक्ट ओरिएंटेड प्रोग्रामिंग भाषा आहे.
ऑब्जेक्ट ओरिएंटेड प्रोग्रामिंगची मुख्य तत्त्वे:

  • ऑब्जेक्ट्स आणि क्लासेस
  • विरासत
  • पॉलीमॉर्फिझम
  • अ‍ॅब्स्ट्रॅक्शन
  • एन्कॅप्सुलेशन
ऑब्जेक्ट्स आणि क्लासेस

ऑब्जेक्ट ओरिएंटेड प्रोग्रामिंग – नावाप्रमाणेच, प्रोग्रामिंगमध्ये ऑब्जेक्ट्सचा वापर करते. ऑब्जेक्ट ओरिएंटेड प्रोग्रामिंगचा उद्देश वास्तविक जगातील घटक जसे की विरासत, लपवणे, पॉलीमॉर्फिझम इत्यादी प्रोग्रामिंगमध्ये अंमलात आणणे आहे. OOP चे मुख्य उद्दीष्ट म्हणजे डेटा आणि त्या डेटा वर कार्य करणाऱ्या फंक्शन्सना एकत्र बांधणे, ज्यामुळे इतर कोड त्या डेटाला प्रवेश करू शकत नाही.

ऑब्जेक्ट : ऑब्जेक्ट म्हणजे काही वैशिष्ट्ये आणि वागणूक असलेली ओळखता येण्याजोगी घटक. ऑब्जेक्ट म्हणजे क्लासचा एक उदाहरण. जेव्हा क्लास परिभाषित केला जातो, तेव्हा कोणतीही मेमरी अलॉट होत नाही, परंतु जेव्हा तो इंस्टंटिएट केला जातो (म्हणजे ऑब्जेक्ट तयार केला जातो), तेव्हा मेमरी अलॉट केली जाते.
क्लास : C++ मध्ये ऑब्जेक्ट ओरिएंटेड प्रोग्रामिंगला कारणीभूत असलेला मुख्य घटक म्हणजे क्लास. तो एक वापरकर्त्याने परिभाषित केलेला डेटा प्रकार आहे, ज्यात त्याचे स्वतःचे डेटा मेंबर्स आणि मेंबर फंक्शन्स असतात, जे त्या क्लासचा एक उदाहरण तयार करून प्रवेश आणि वापरता येतात. क्लास म्हणजे ऑब्जेक्टसाठी ब्लूप्रिंटसारखा आहे. उदाहरणार्थ: कारचा क्लास विचार करा. वेगवेगळ्या नावाच्या आणि ब्रँडच्या अनेक कार्स असू शकतात, परंतु त्या सर्वांमध्ये काही सामान्य वैशिष्ट्ये असतात जसे की सर्वांमध्ये ४ चाके, स्पीड लिमिट, मायलेज रेंज इत्यादी. येथे, कार हा क्लास आहे आणि चाके, स्पीड लिमिट, मायलेज ही त्याची वैशिष्ट्ये आहेत.

विरासत

विरासत म्हणजे एक क्लास दुसऱ्या क्लासकडून गुणधर्म आणि वैशिष्ट्ये मिळवण्याची क्षमता. विरासत ही ऑब्जेक्ट ओरिएंटेड प्रोग्रामिंगची एक अत्यंत महत्त्वाची वैशिष्ट्य आहे.
सब क्लास : दुसऱ्या क्लासकडून गुणधर्म मिळवणारा क्लास म्हणजे सब क्लास किंवा डेरिव्हड क्लास.
सुपर क्लास : ज्याच्या गुणधर्मांचा सब क्लासकडून उपयोग केला जातो, तो क्लास म्हणजे बेस क्लास किंवा सुपर क्लास. विरासत वापरून, आपल्याला फंक्शन्स फक्त एकदाच लिहावी लागतात, कारण आपण बेस क्लासकडून इतर तीन क्लासेस वारसा घेतले आहेत.

विरासतचे प्रकार :

सिंगल विरासत : सिंगल विरासत मध्ये, एक क्लास फक्त एका क्लासकडूनच गुणधर्म मिळवू शकतो.

मल्टिपल विरासत : मल्टिपल विरासत मध्ये, एक क्लास एकापेक्षा जास्त क्लासेसकडून गुणधर्म मिळवू शकतो.

मल्टीलिव्हल विरासत : या प्रकारात, एक डेरिव्हड क्लास दुसऱ्या डेरिव्हड क्लासकडून गुणधर्म मिळवतो.

हायरेर्किकल विरासत : या प्रकारात, एक बेस क्लासकडून अनेक डेरिव्हड क्लासेस गुणधर्म मिळवतात.

हायब्रीड (व्हर्च्युअल) विरासत : हायब्रीड विरासत म्हणजे दोन किंवा अधिक प्रकारच्या विरासतचे संयोजन. उदाहरणार्थ: हायरेर्किकल विरासत आणि मल्टिपल विरासतचे संयोजन.

पॉलीमॉर्फिझम

पॉलीमॉर्फिझम म्हणजे "अनेक रूपे" असणे. साध्या शब्दांत, पॉलीमॉर्फिझम म्हणजे संदेश विविध रूपांत प्रदर्शित करण्याची क्षमता. वास्तविक जीवनातील उदाहरण: एक व्यक्ती एकाच वेळी वडील, पती, कर्मचारी अशा विविध भूमिकांमध्ये असू शकतो. म्हणजेच, तीच व्यक्ती विविध परिस्थितींमध्ये विविध वागणूक दाखवते. हेच पॉलीमॉर्फिझम आहे.
C++ मध्ये पॉलीमॉर्फिझम मुख्यतः दोन प्रकारांमध्ये विभागला जातो:
१. संकलन काल पॉलीमॉर्फिझम
२. रनटाइम पॉलीमॉर्फिझम

संकलन काल पॉलीमॉर्फिझम: हा प्रकार फंक्शन ओव्हरलोडिंग किंवा ऑपरेटर ओव्हरलोडिंगद्वारे साधता येतो.
रनटाइम पॉलीमॉर्फिझम: हा प्रकार फंक्शन ओव्हररायडिंगद्वारे साधता येतो.

अ‍ॅब्स्ट्रॅक्शन

डेटा अ‍ॅब्स्ट्रॅक्शन ही ऑब्जेक्ट ओरिएंटेड प्रोग्रामिंगमधील एक अत्यंत महत्त्वाची वैशिष्ट्य आहे. अ‍ॅब्स्ट्रॅक्शन म्हणजे फक्त आवश्यक माहिती प्रदर्शित करणे आणि तपशील लपवणे. डेटा अ‍ॅब्स्ट्रॅक्शन म्हणजे बाह्य जगाला फक्त आवश्यक माहिती प्रदान करणे आणि अंतर्गत तपशील किंवा अंमलबजावणी लपवणे.
